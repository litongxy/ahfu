# 小程序免密登录接入说明

## 1. 后端配置

在 `acp-gateway` 的运行环境里配置以下变量：

```bash
WECHAT_APP_ID=你的小程序AppID
WECHAT_APP_SECRET=你的小程序AppSecret
AUTH_TOKEN_SECRET=用于签名Token的随机长字符串
AUTH_TOKEN_EXPIRES_SECONDS=604800

# 可选：是否强制所有业务接口必须登录
AUTH_STRICT_MODE=true

# 可选：生产环境默认禁用，设置 true 可开启调试登录接口
ENABLE_DEV_LOGIN=false
```

## 2. 登录接口

### 小程序登录

- `POST /acp/auth/mini-program/login`
- 入参：

```json
{
  "code": "wx.login 返回的临时 code"
}
```

- 返回：

```json
{
  "code": 0,
  "data": {
    "token": "Bearer 使用的 token",
    "expiresInSeconds": 604800,
    "user": {
      "userId": "wx_openid",
      "openId": "openid",
      "unionId": "unionid(可选)"
    }
  }
}
```

### 调试登录（开发态）

- `POST /acp/auth/dev-login`
- 入参可选：

```json
{
  "openId": "自定义调试openid"
}
```

## 3. 小程序端调用示例

```js
function login() {
  return new Promise((resolve, reject) => {
    wx.login({
      success: ({ code }) => {
        if (!code) {
          reject(new Error('wx.login 未返回 code'));
          return;
        }

        wx.request({
          url: `${BASE_URL}/acp/auth/mini-program/login`,
          method: 'POST',
          data: { code },
          success: (res) => {
            if (res.statusCode !== 200 || res.data.code !== 0) {
              reject(new Error(res.data.message || '登录失败'));
              return;
            }

            const token = res.data.data.token;
            wx.setStorageSync('access_token', token);
            resolve(res.data.data.user);
          },
          fail: reject,
        });
      },
      fail: reject,
    });
  });
}
```

## 3.1 当前仓库的推荐接法：小程序壳 + H5 `web-view`

这个仓库里的 `miniprogram-shell` 不是原生小程序页面直连业务接口，而是：

- 小程序页面先打开一个 `web-view`
- `web-view` 里再加载 `chat.html`

这种场景下，`chat.html` 里的 H5 代码通常拿不到真正的小程序运行时 `wx.login()` 能力，所以不要把自动登录放在 H5 里做。更稳的链路是：

1. 小程序壳页调用 `wx.login()`
2. 小程序壳页请求 `/acp/auth/mini-program/login`
3. 服务端返回业务 token
4. 小程序壳页把 token 通过 `web-view src` 传给 H5
5. H5 启动时消费这个 token，写入本地存储，后续请求统一带 `Authorization`

当前仓库已经按这个方式补好了这几个点：

- 小程序壳登录页：[miniprogram-shell/pages/index/index.js](/Users/litongb/dev/ahfu2/miniprogram-shell/pages/index/index.js)
- H5 鉴权桥接：[scripts/auth-client.js](/Users/litongb/dev/ahfu2/scripts/auth-client.js)

小程序壳页核心逻辑：

```js
wx.login({
  success: ({ code }) => {
    wx.request({
      url: `${API_BASE_URL}/acp/auth/mini-program/login`,
      method: 'POST',
      data: { code },
      success: (res) => {
        const token = res.data.data.token;
        const url = `${CHAT_URL}#mp_token=${encodeURIComponent(token)}`;
        this.setData({ url });
      },
    });
  },
});
```

H5 侧启动时消费 token：

```js
const hashParams = new URLSearchParams(window.location.hash.slice(1));
const token = hashParams.get('mp_token');

if (token) {
  localStorage.setItem('acp_auth_token', token);
  hashParams.delete('mp_token');
  history.replaceState({}, '', `${location.pathname}${location.search}${hashParams.toString() ? `#${hashParams}` : ''}`);
}
```

## 4. 业务接口鉴权

后续调用业务接口时，带上 `Authorization`：

```js
function requestWithAuth(options) {
  const token = wx.getStorageSync('access_token');
  return wx.request({
    ...options,
    header: {
      ...(options.header || {}),
      Authorization: `Bearer ${token}`,
    },
  });
}
```

## 5. 当前项目已接入的鉴权行为

- 聊天：`/acp/chat`、`/acp/chat/stream`
- 健康档案：`/acp/health/profile`
- 报告：`/acp/report/*`
- 有 token 时会自动按 token 中的 `userId` 做用户隔离，防止跨用户读写数据。
- 当 `AUTH_STRICT_MODE=true` 时，上述业务接口会强制登录后才能访问。

## 6. 对接时你还需要确认的配置

- 小程序后台要把网关域名加入“`request 合法域名`”
- `web-view` 打开的 H5 域名要加入“`业务域名`”
- 后端环境变量里要配置 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET`
- 生产环境建议配置强随机 `AUTH_TOKEN_SECRET`
- 如果你把 H5 入口地址从 `http://127.0.0.1:3000/chat.html` 改成正式域名，同步修改 [miniprogram-shell/pages/index/index.js](/Users/litongb/dev/ahfu2/miniprogram-shell/pages/index/index.js) 顶部的 `CHAT_URL`
