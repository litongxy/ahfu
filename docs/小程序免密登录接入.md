# 小程序免密登录接入说明

## 1. 后端配置

在 `acp-gateway` 的运行环境里配置以下变量：

```bash
WECHAT_APP_ID=你的小程序AppID
WECHAT_APP_SECRET=你的小程序AppSecret
AUTH_TOKEN_SECRET=用于签名Token的随机长字符串
AUTH_TOKEN_EXPIRES_SECONDS=604800

# 可选：是否强制所有业务接口必须登录
AUTH_STRICT_MODE=true

# 可选：生产环境默认禁用，设置 true 可开启调试登录接口
ENABLE_DEV_LOGIN=false
```

## 2. 登录接口

### 小程序登录

- `POST /acp/auth/mini-program/login`
- 入参：

```json
{
  "code": "wx.login 返回的临时 code"
}
```

- 返回：

```json
{
  "code": 0,
  "data": {
    "token": "Bearer 使用的 token",
    "expiresInSeconds": 604800,
    "user": {
      "userId": "wx_openid",
      "openId": "openid",
      "unionId": "unionid(可选)"
    }
  }
}
```

### 调试登录（开发态）

- `POST /acp/auth/dev-login`
- 入参可选：

```json
{
  "openId": "自定义调试openid"
}
```

## 3. 小程序端调用示例

```js
function login() {
  return new Promise((resolve, reject) => {
    wx.login({
      success: ({ code }) => {
        if (!code) {
          reject(new Error('wx.login 未返回 code'));
          return;
        }

        wx.request({
          url: `${BASE_URL}/acp/auth/mini-program/login`,
          method: 'POST',
          data: { code },
          success: (res) => {
            if (res.statusCode !== 200 || res.data.code !== 0) {
              reject(new Error(res.data.message || '登录失败'));
              return;
            }

            const token = res.data.data.token;
            wx.setStorageSync('access_token', token);
            resolve(res.data.data.user);
          },
          fail: reject,
        });
      },
      fail: reject,
    });
  });
}
```

## 4. 业务接口鉴权

后续调用业务接口时，带上 `Authorization`：

```js
function requestWithAuth(options) {
  const token = wx.getStorageSync('access_token');
  return wx.request({
    ...options,
    header: {
      ...(options.header || {}),
      Authorization: `Bearer ${token}`,
    },
  });
}
```

## 5. 当前项目已接入的鉴权行为

- 聊天：`/acp/chat`、`/acp/chat/stream`
- 健康档案：`/acp/health/profile`
- 报告：`/acp/report/*`
- 有 token 时会自动按 token 中的 `userId` 做用户隔离，防止跨用户读写数据。
- 当 `AUTH_STRICT_MODE=true` 时，上述业务接口会强制登录后才能访问。
