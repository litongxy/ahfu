# 同仁堂元气助手 - 产品设计文档

**版本**：V1.0  
**编制日期**：2026年3月  
**产品名称**：同仁堂元气助手  
**产品类型**：AI健康助手应用（APP/小程序）

---

## 1. 产品概述

### 1.1 产品背景

随着人们对健康管理的重视程度不断提升，传统的健康服务模式已难以满足用户日益增长的个性化需求。同仁堂作为中医药行业的领军品牌，拥有丰富的健康知识资源和品牌影响力。基于AI技术打造一款智能健康助手应用，可以为用户提供便捷、专业的健康咨询服务，同时深度融合同仁堂的商品和内容资源，实现用户体验与商业价值的双赢。

### 1.2 产品定位

**一款基于AI技术的智能健康助手**，为用户提供健康咨询、体检报告分析、健康内容推荐、健康商品推荐等一站式健康管理服务。

### 1.3 产品目标

1. **用户目标**
   - 快速获取专业、准确的健康知识解答
   - 便捷地上传和分析体检报告
   - 获得个性化的健康干预方案
   - 发现适合的健康内容和商品

2. **业务目标**
   - 提升用户健康管理的便捷性和专业性
   - 增加用户对同仁堂品牌的认知和信任
   - 实现健康内容和商品的精准推荐
   - 构建完整的用户健康数据体系

### 1.4 目标用户

| 用户类型 | 特征描述 | 核心需求 |
|---------|----------|----------|
| 25-35岁职场人群 | 工作压力大，关注亚健康问题 | 快速获取健康建议、缓解工作压力 |
| 35-50岁中年人群 | 体检需求增多，关注慢性病 | 体检报告解读、慢病管理 |
| 50岁以上人群 | 注重养生，关注中医药 | 中医养生指导、长期健康档案 |
| 健康意识强的人群 | 主动管理健康，消费能力强 | 个性化健康方案、优质健康产品 |

## 4. 核心功能详细设计

### 4.1 健康咨询

#### 4.1.1 功能概述

用户通过主页聊天输入框发起健康相关咨询，AI根据知识库和用户意图进行智能回复，并推荐相应的健康内容或服务。

#### 4.1.2 支持的场景

| 场景 | 子场景 | 功能描述 |
|------|--------|----------|
| 饮食健康 | 食谱推荐、饮食建议 | 根据用户饮食目标推荐食谱，给出饮食建议 |
| 运动健身 | 课程推荐、运动建议 | 根据用户运动目标推荐健身视频，给出运动建议 |
| 心理健康 | 心理安抚、课程推荐 | 根据用户问题给出建议，推荐心理课程，AI心理咨询师 |
| 睡眠健康 | 睡眠建议、课程推荐 | 根据用户问题给出建议，推荐睡眠改善课程 |
| 减龄抗衰 | 抗衰建议、产品推荐 | 根据用户问题给出建议，推荐抗衰项目/产品 |
| 疾病防控 | 疾病咨询、预防建议 | 根据用户问题给出疾病防控建议 |
| 中医养生 | 中医问答、体质测试 | 根据用户问题给出中医建议，引导用户进行体质测试 |
| 同仁堂文化 | 品牌问答 | 回答关于同仁堂的历史、产品、文化等问题 |

#### 4.1.3 用户交互流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户输入   │ ──→ │  意图识别   │ ──→ │  场景匹配   │
└─────────────┘     └─────────────┘     └─────────────┘
                                                 │
                     ┌─────────────┐     ┌───────┴───────┐
                     │  推荐内容   │ ←── │  知识库检索   │
                     │  /商品/页面 │     └───────────────┘
                     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  用户反馈   │
                    │  (可选)    │
                    └─────────────┘
```

#### 4.1.4 精准健康问卷

为提供更精准的服务，AI可引导用户填写精准健康问卷，包括：

- 基本信息：年龄、性别、身高、体重
- 饮食习惯：饮食偏好、过敏原、三餐规律
- 运动习惯：运动频率、偏好运动类型
- 睡眠情况：睡眠质量、睡眠时长、入睡时间
- 压力状况：工作压力、生活压力自评
- 健康目标：减重、增肌、改善睡眠、缓解压力等

### 4.2 报告分析

#### 4.2.1 功能概述

用户通过上传体检报告，AI智能分析报告数据，识别异常项，并给出个性化的复查建议和调理方案。

#### 4.2.2 功能流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  报告上传   │ ──→ │  数据解析   │ ──→ │  异常识别   │
└─────────────┘     └─────────────┘     └─────────────┘
                                               │
                     ┌─────────────┐     ┌───────┴───────┐
                     │  干预方案   │ ←── │  问卷检查    │
                     │  推荐      │     └───────────────┘
                     └─────────────┘
                           │
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
    │  饮食方案   │  │  运动方案   │  │  睡眠方案   │
    │  (食谱推荐) │  │  (课程推荐) │  │  (课程推荐) │
    └─────────────┘  └─────────────┘  └─────────────┘
```

#### 4.2.3 报告分析结果页

**有异常情况**：
- 显示异常项数量（如：3项异常）
- 异常项详细列表：
  - 异常名称（如：轻度脂肪肝）
  - 详细描述（如：B超提示）
  - 复查建议（如：建议复查）
- 总结建议：
  - 异常原因分析
  - 复查时间建议
  - 具体复查项目
  - 生活方式调整建议
- 个性化调理方案：
  - 推荐食谱（含卡路里）
  - 推荐运动课程
  - 推荐睡眠/心理课程

**无异常情况**：
- 显示"您的体检报告有0项异常"
- 身体健康确认提示
- 保持健康建议
- 精准健康入口（引导完善资料）

#### 4.2.4 数据字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| report_id | string | 报告ID |
| user_id | string | 用户ID |
| upload_time | datetime | 上传时间 |
| report_data | json | 报告原始数据 |
| anomaly_items | array | 异常项列表 |
| analysis_result | string | 分析结果 |
| intervention_plan | json | 干预方案 |

### 4.3 内容推荐

#### 4.3.1 功能概述

根据用户咨询内容和健康档案，智能推荐相关的健康内容，包括食谱、运动课程、心理课程、睡眠课程等。

#### 4.3.2 推荐流程

```
用户咨询 → 意图识别 → 内容匹配 → 个性化排序 → 推荐展示
```

#### 4.3.3 内容类型

| 内容类型 | 示例 | 推荐场景 |
|----------|------|----------|
| 食谱 | 牛肉土豆杂蔬焖菜、玉米鸡胸肉蔬菜焖面 | 饮食咨询、体检异常 |
| 运动课程 | HIIT燃脂、零基础健身 | 运动咨询、体检异常 |
| 心理课程 | 7天正念减压 | 心理咨询、睡眠问题 |
| 睡眠课程 | 7天轻松入睡 | 睡眠咨询 |

### 4.4 商品推荐

#### 4.4.1 功能概述

根据用户咨询内容和健康需求，智能推荐相关的健康商品，如保健品、食品、器材等。

#### 4.4.2 推荐策略

- 基于用户咨询意图匹配商品
- 基于体检异常项推荐相关商品
- 基于用户健康档案推荐商品
- 基于同类用户购买数据推荐

### 4.5 功能页路由

#### 4.5.1 功能概述

识别用户语义中的功能查询意图，快速跳转到对应的APP功能页面。

#### 4.5.2 支持的功能页面

- 体检报告分析页
- 精准健康页
- 食谱列表页
- 运动课程页
- 心理课程页
- 商品详情页
- 订单页
- 门店页
- 用户档案页

## 6. 后台管理系统设计

### 6.1 系统角色

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 全部权限 |
| 内容运营 | 内容管理、热门问题管理 |
| 商品运营 | 商品库管理 |
| 客服 | AI沟通明细查看 |

### 6.2 功能模块

#### 6.2.1 AI场景管理

功能：管理AI对话场景配置

| 字段 | 类型 | 说明 |
|------|------|------|
| 场景名称 | string | 如：饮食健康、运动健身 |
| 场景定义 | text | 场景描述和边界 |
| 创建时间 | datetime | 创建时间 |
| 创建人 | string | 创建人 |

操作：新增、查看、编辑、删除

#### 6.2.2 知识库管理

功能：管理AI知识库问答对

| 字段 | 类型 | 说明 |
|------|------|------|
| 问题 | string | 标准问题 |
| 答案 | text | 标准答案 |
| 关联场景 | string | 所属场景 |

操作：新增、编辑、删除

#### 6.2.3 热门问题管理

功能：配置首页快捷问题

| 字段 | 类型 | 说明 |
|------|------|------|
| 问题 | string | 问题内容 |
| 关联场景 | string | 关联场景 |
| 排序 | int | 显示顺序 |
| 启用状态 | boolean | 是否启用 |
| 点击PV | int | 浏览量 |
| 点击UV | int | 点击用户数 |

#### 6.2.4 内容管理

功能：管理食谱、运动课程、心理课程、睡眠课程

| 内容类型 | 字段 |
|----------|------|
| 食谱 | 食谱名称、食谱分类、描述、卡路里、对应场景目标、启用状态 |

Request:
{
  "user_id": "xxx",
  "message": "用户问题",

}

Response:
{
  "code": 0,
  "data": {
    "response": "AI回复内容",
    "scene": "场景类型",
    "recommendations": {
      "contents": [],
      "goods": [],
      "pages": []
    }

Request:

Response:
{
  "code": 0,
  "data": {
    "anomaly_items": [],

  }

  }
}
